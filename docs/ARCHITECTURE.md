# Архитектура проекта (актуально)

Проект — одностраничная web-игра (Canvas 2D), **ES Modules**, без сборки и без внешних зависимостей/ассетов.

## 1) Слои

### `index.html`
- Подключает `src/main.js`
- Содержит корневые контейнеры: `canvas` и DOM для HUD/модалок

### `src/main.js`
- Создает `Game` и запускает цикл

### `src/core/*` — движок
- `game.js` — сборка сервисов, регистрация состояний FSM, главный цикл
- `stateMachine.js` — простая FSM (enter/update/render/exit)
- `input.js` — мышь/тач/клавиатура (one-frame flags)
- `audio.js` — WebAudio (генерируемые звуки), настройки громкости/выкл.

### `src/ui/*` — UI состояния и оверлеи
- `titleState.js`, `gameOverState.js`, `victoryState.js`
- `overlays.js` — HTML справки/настроек, бинды для настроек

### `src/gameplay/*` — правила и игровые состояния
- `model.js` — структура сохранения + нормализация при загрузке
- `storage.js`, `prefs.js` — localStorage (сейв и настройки отдельно)
- `worldGen.js` — генерация карты и объектов
- `overworldState.js` — исследование (шаг на соседнюю клетку, вход в события/бой/лагерь)
- `eventState.js` — алтарь/руины/аномалия (логика + рисованные анимации)
- `campState.js` — лечение/найм/бафы
- `echoState.js` — «Эхо времени»
- `battleState.js` — строго пошаговый бой (движение + действие) + анимация перемещения
- `postBattleState.js` — экран итогов боя (награды, переходы)
- `worldRules.js` — баланс (силы врагов, награды, эффекты боя)

### `src/render/*` — рендеринг
- `gridSquare.js` — квадратная карта (Overworld)
- `hexMath.js` — гексы (Battle)
- `unitSprites.js` — рисованные «спрайты» юнитов на Canvas
- `icons.js` — рисованные иконки объектов карты (без ассетов)

## 2) Принципы
- **Данные отдельно от UI:** модель хранится в `model.js`, UI состояния читают/меняют её через правила.
- **Один источник истины:** сохранение модели в localStorage (ключ `logres_save_v1`).
- **Без зависимостей:** все визуалы/анимации — процедурное рисование Canvas.
